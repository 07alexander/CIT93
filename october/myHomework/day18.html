Section 4 Lecture 35 - Function Statements and Function Expressions

  • Expression: A unit of code that results in a value; It doesn't have to save to a variable
  • An anonymous function is a function that doesn't have a name in it's name property
  • An anonymous function cannot be hoisted since the function is being passed into a variable
  • When passing a function to a function, to invoke the function you simply add parenthesis

Coding Along:

    greet();

function greet() {
    console.log('hi');
}

var anonymousGreet = function() {
    console.log('hi');
}

anonymousGreet();

function log(a) {
   a();
}

log(function() {
    console.log('hi');
});

Section 4 Lecture 36 - Conceptual Aside: By Value vs By Reference

  • By value, passing or referencing, or setting equal one value to another, by copying the value
  • By reference, behaves differently from by value, all objects interact by reference, when sending them equal to each other
  • Parameters, objects are passed by reference, primitives by value but objects by reference
  • In javascript you don't have the option to use code to choose if something is passed by value or reference with your code syntax

Coding Along:

    // by value (primitives)
var a = 3;
var b;

b = a;
a = 2;

console.log(a);
console.log(b);

// by reference (all objects (including functions))
var c = { greeting: 'hi' };
var d;

d = c;
c.greeting = 'hello'; // mutate

console.log(c);
console.log(d);

// by reference (even as parameters)
function changeGreeting(obj) {
    obj.greeting = 'Hola'; // mutate
}

changeGreeting(d);
console.log(c);
console.log(d);

// equals operator sets up new memory space (new address)
c = { greeting: 'howdy' };
console.log(c);
console.log(d);

Section 4 Lecture 37 - Objects, Functions, and 'this'

  • In javascript engine every time and execution context is created it gives us this variable called 'this'
  • If a value of an object is a function, it is called a method
  • You can reference the method by calling the var along with the dot operator
  • When you use 'this' within an object, it will reference the object that contains 'this'

Coding Along:

    function a() {
    console.log(this);
    this.newvariable = 'hello';
}

var b = function() {
    console.log(this);
}

a();

console.log(newvariable); // not good!

b();

    name: 'The c object',

        console.log(self);

        var setname = function(newname) {
            self.name = newname;

Section 4 Lecture 38 - Conceptual Aside: Arrays - Collections of Anything

  • An array is a collection that holds many things inside of it
  • Arrays can store numbers, booleans, objects, function expression, and strings
  • The function in an array is created on the fly so it becomes an expression rather than function statement
  • You can reference the elements in the array using a 0-based system, so first element is zero , second element being 1 and so on...

    "hello"

Section 4 Lecture 39 - 'arguments' and spread

  • Arguments: The parameters you pass to a function; Javascript gives you a keyword of the same name which contains them all.
  • You can skip the passing of parameters or pass on part of the parameters in javascript
  • Special keyword, argument, contains a list of parameters that you have passed
  • Spread parameter, parameters that are passing to a function, you can add a parameter with a ...'name', it will wrap up other parameters

Coding Along:

    function greet(firstname, lastname, language) {

    language = language || 'en';

    if (arguments.length === 0) {
        console.log('Missing parameters!');
        console.log('-------------');
        return;
    }

    console.log(firstname);
    console.log(lastname);
    console.log(language);
    console.log(arguments);
    console.log('arg 0: ' + arguments[0]);
    console.log('-------------');

}
