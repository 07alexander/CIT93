Section 4 Lecture 30 - Objects and the Dot

  • Object will contain Primitive(property), object(property), and function(method)
  • The dot is an operator that has member access to the properties of an object
  • The dot operator is a left-to-right operator, so you can keep storing properties from left to right
  • The brackets can also be used in the same way as the dot operator to store objects.

Coding Along:

    var person = new Object();

person["firstname"] = "Tony";
person["lastname"] = "Alicea";

var firstNameProperty = "firstname";

console.log(person);
console.log(person[firstNameProperty]);

console.log(person.firstname);
console.log(person.lastname);

person.address = new Object();
person.address.street = "111 Main St.";
person.address.city = "New York";
person.address.state = "NY";

console.log(person.address.street);
console.log(person.address.city);
console.log(person["address"]["state"]);

Section 4 Lecture 31 - Objects and Object Literals

  • When the Javascript engine is parsing curly braces and it's not part of something like an if statement or a for loop, it assumes you're creating an object
  • Using the curly braces for creating an object, makes it easier and more human-readable
  • Using object literals allows javascript to create an object on-the-fly within functions parameters
  • The physical code that you're writing isn't really happening under the hood, it's being translated to something the javascript engine can understand

Coding Along:

    var Tony = {
    firstname: 'Tony',
    lastname: 'Alicea',
    address: {
        street: '111 Main St.',
        city: 'New York',
        state: 'NY'
    }
};

function greet(person) {
    console.log('Hi ' + person.firstname);
}

greet(Tony);

greet({
    firstname: 'Mary',
    lastname: 'Doe'
});

Tony.address2 = {
    street: '333 Second St.'
}

Section 4 Lecture 32 - Framework Aside: Faking Namespaces

  • Namespace: A container for variables and functions
  • Typically to keep variables and functions with the same name separate
  • We can prevent a name collision by creating an object that will be the container for our properties amd methods and the things we want to use
  • The dot operator does creation one at a time, so it will set to undefined if object is empty

Coding Along:

    var greet = 'Hello!';
    var greet = 'Hola!';

    console.log(greet);

    var english = {
        greetings: {
            basic: 'Hello!'
        }
    };

    var spanish = {};

    spanish.greet = 'Hola!';

    console.log(english);

Section 4 Lecture 33 - JSON and Object Literals

  • JSON: JavaScript Object Notation, inspired by object literal syntax in javascript
  • JSON isn't really a part of javascript, but because it's so popular and so easy for javascript to understand what this is,
  • Javascript does come with some built in functionality to transfer between the two.
  • It has made for a really easy transition when pushing data from the client in a browser to the server

Coding Along:

    var objectLiteral = {
    firstname: 'Mary',
    isAProgrammer: true
}

console.log(JSON.stringify(objectLiteral));

var jsonValue = JSON.parse('{ "firstname": "Mary", "isAProgrammer": true }');

console.log(jsonValue);
