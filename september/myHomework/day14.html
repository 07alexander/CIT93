Section 3 Lecture 25 - Comparison Operators

  • Javascript engine will convert boolean into numbers if it has to (coercing a data type when interpreting)
  • When trying to coerce "undefined" Javascript will spit out NaN(not a number)
  • Use triple equals when making equality comparisons, unless you explicitly want to coerce the two value
  • As default make sure to always use triple equals for equality comparisons

Coding Along:

    console.log(1<2<3); //true
    console.log(3<2<1); //true
    console.log(false < 1); //Converts boolean into a number
    var a = 0;

    if(a == b) { // They are equal, if use "===" then nope not equal
        console.log("They are equal!");
    }else{
        console.log("Nope, not equal.");
    }

Section 3 Lecture 26 - Equality Comparisons Table

  • There are four equality algorithms in ES2015
  • Abstract Equality Comparison (==)
  • Strict Equality Comparison (===): used by Array.prototype.indexOf, Array.prototype.lastIndexOf, and case-matching
  • SameValueZero: used by %TypedArray% and ArrayBuffer constructors, as well as Map and Set operations, and up-coming String.prototype.includes in ES2016
  • SameValue: used in all other places

Section 3 Lecture 27 - Existence and Booleans

  • When trying to convert undefined to a boolean, undefined becomes false when it's converted in the JavaScript engine
  • Whatever you put inside the parentheses of an if statement, it will attempt to convert or coerce to a boolean
  • We can use coercion to our own advantage and check to see if a variable has a value
  • This pattern of coercion, of using coercion in this case, to check for existence is used in a lot of frameworks and libraries

Coding Along:

    var a;
    //goes to internet and looks for a value
    a="hi"; //Something is there
    for(a){
        console.log('Something is there.');
    }

Section 3 Lecture 28 - Default Values

  • Invocation is initially set when the memory space is set up to undefined
  • Javascript simply ignores function calls without the expected parameters and simply says it's okay
  • Operators are functions that return values, that's why name is equal to name, the value that currently has a default value or input value
  • You can set a default value with a one line OR expression
    function greet(name) {
    name = name || 'Your name here';
    console.log('Hello ' + name);
}

greet('Tony'); //Hello Tony
greet(); //Hello Your name here
>

Section 3 Lecture 29 - Framework Aside: Default Values

  • For our purposes, a framework and a library are the same thing; a grouping of javascript code that performs a task and and is intended to be reusable
  • Script tags do not create new execution context, they are simply stacked onto each other
  • You to make sure your libraries don't collide when you're stacking them or else you might get undesirable results
  • So when you look inside the source code you might see code that checks for the global namespace or object so it doesn't collide or override it 
    window.libraryName = window.libraryName || "Lib2"; // Checking for name collision in libraries
