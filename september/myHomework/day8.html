Section 2 Lecture 9 - The Global Environment and The Global Object

  • Execution Context: Your base execution context is the global execution context.
  • Global execution context creates a global object and 'this'.
  • The javascript engine will create 'this' even when you haven't written any code
  • Global: not an inside function

Section 2 Lecture 10 - The Global Environment, Execution Context, Creation and Hoisting

  • Hoisting: variables and function are moved to the top by the JavaScript engine as if they were moved physically up to the top
  • The execution context is created in two phases; Creation Phase and Execution Phase
  • The Creation Phase: Setting up Global object, 'this', Outer Environment, and it also sets up memory space for variable and functions for "Hoisting"
  • The JavaScript Engine in the Execution Phase likes to put placeholders to make all variables initially undefined.
  • The JavaScript Engine translates my code and then make decisions on what to do what that code instead of directly executing my code

Section 2 Lecture 11 - Conceptual Aside: Javascript and 'undefined'

  • Undefined and not defined are two different things
  • Undefined actually means that the variable hasn't actually been set
  • Not defined is when a variable you are trying to reference is not declared in memory
  • Setting values to undefined manually is bad because it'll make debugging harder ;you won't know whether you set it to undefined or if the engine did

Section 2 Lecture 12 - The Execution Context - Code Execution

  • The execution phase, when it's time to run your code, line by line
  • Code Execution happens after the first phase, the Creation Phase
  • This is when the engine is interpreting it, converting it, compiling it and executing it on the computer.

Section 2 Lecture 13 - Conceptual Aside: Single Threaded, Synchronous Execution

  • Single Threaded: One command at a time; under the hood of the browser, maybe not.
  • JavaScript behaves in a single threaded manner
  • Synchronous: One at a time; and in order...
  • The code is executed one line at time in the order they appear
